# A11y Baseline — 2026-04-27

First run of `/audit-a11y` against main. All routes use the test PocketBase
container (port 28090). `/team` skipped — no approved coach seeded in the
test PB.

## Run conditions

- Axe tags: `wcag2a`, `wcag2aa`, `wcag21aa`, `wcag22aa`
- Browser: Chromium (headless)
- Server: SvelteKit preview build + test PB on port 28090
- Svelte-check: 0 errors, 0 warnings (4595 files)
- Static grep: no violations

## Findings

### Critical

- **Missing `<title>` element on every audited route** (`web/src/app.html`
  likely lacks a static title and no route adds `<svelte:head><title>`).
  Axe `serious`, WCAG 2.4.2 (Level A). Affects all 7 scanned routes.
  Screen reader and tab-switcher users cannot identify the page.

- **Three unlabeled number inputs in the "Create Tournament" form** at
  `web/src/routes/admin/tournaments/+page.svelte` (`year`, line ~124;
  `num_preliminary_rounds`, line ~134; `num_elimination_rounds`, line ~145).
  Axe `critical`, WCAG 4.1.2. Fields sit in `<Table.Cell>` with column
  headers but have no `aria-label`, `aria-labelledby`, wrapping `<label>`,
  or placeholder. AT users hear only "edit, number."

### Warnings

None.

### Suggestions

- `web/src/routes/admin/tournaments/+page.svelte:115` — the tournament name
  input uses `placeholder="Tournament name"` as its only visible label. Axe
  did not flag it, but WCAG 3.3.2 wants a persistent label (placeholder
  vanishes on focus and has poor contrast in many UAs).

### Praise

- No `div`/`span` click handlers, no `javascript:` hrefs, no `<a>` without
  `href`, no missing `alt`, no unlabeled visible inputs outside the
  tournament form per static grep.
- All `outline-none` usages in shadcn-svelte components pair with
  `focus-visible:ring-3 focus-visible:ring-ring/50` — focus rings are
  preserved.
- `svelte-check` reports 0 a11y warnings across 4595 files.
- 6 of 7 audited routes would pass axe entirely once `<title>` is added.
